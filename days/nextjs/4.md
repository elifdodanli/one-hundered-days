## ğŸ“– Task Explanations

Next.js offers different rendering strategies to optimize your application for performance and SEO. With Static Site Generation (SSG), the HTML is generated at build time and can be served from a CDN, making it incredibly fast. With Server-Side Rendering (SSR), the HTML is generated on the server for each request, which is useful for pages with frequently changing data. Today, you will learn how to implement both using `getStaticProps` and `getServerSideProps`.

## ğŸ“š Dictionary for Developers

*   **Static Site Generation (SSG):** The process of generating the HTML for a page at build time.
*   **Server-Side Rendering (SSR):** The process of generating the HTML for a page on the server for each request.
*   **`getStaticProps`:** A Next.js function that you can export from a page to fetch data and pre-render the page at build time.
*   **`getServerSideProps`:** A Next.js function that you can export from a page to fetch data and render the page on the server for each request.
