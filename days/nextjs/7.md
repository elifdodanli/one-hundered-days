## ğŸ“– Task Explanations

While `getStaticProps` and `getServerSideProps` are great for initial page loads, you will often need to fetch data on the client-side in response to user interactions. Today, you will learn how to do this, first with the standard `useEffect` Hook and then with SWR. SWR (Stale-While-Revalidate) is a powerful library that provides caching, revalidation, and many other features out of the box, making your client-side data fetching much more efficient.

## ğŸ“š Dictionary for Developers

*   **Client-Side Data Fetching:** The process of fetching data from an API in the browser after the initial page load.
*   **SWR (Stale-While-Revalidate):** A strategy to first return the data from cache (stale), then send the fetch request (revalidate), and finally come with the up-to-date data.
*   **`useSWR`:** The primary Hook provided by the SWR library for fetching data.
