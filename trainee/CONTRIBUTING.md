# Contribution Guide

Thank you for your interest in contributing to the MasterFabric Academy!

We welcome contributions from everyone. By participating in this project, you agree to abide by our [Code of Conduct](./CODE_OF_CONDUCT.md).

## How to Contribute

1.  **Find an Issue:** Look for existing issues or create a new one to discuss the changes you'd like to make.
2.  **Fork the Repository:** Create your own copy of the repository to work on.
3.  **Create a Branch:** Create a new branch for your feature or bug fix (`git checkout -b feature/your-feature-name`).
4.  **Make Your Changes:** Write your code or documentation.
5.  **Commit Your Changes:** Follow the commit message guidelines (`git commit -m "feat: Add some amazing feature"`).
6.  **Push to Your Branch:** Push your changes to your forked repository (`git push origin feature/your-feature-name`).
7.  **Open a Pull Request:** Submit a pull request from your branch to the main repository's `main` branch.

## Pull Request Process

1.  Ensure any new code is accompanied by tests.
2.  Update the documentation to reflect your changes.
3.  A project maintainer will review your pull request and provide feedback.

We appreciate your contributions to making software development education more accessible!
